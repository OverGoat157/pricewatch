# План презентации (10–12 слайдов)

Раскрывает 5 вопросов защиты: постановка задачи · выбор инструментов ·
ход решения · результаты · выводы.

1. **Титульный слайд** — тема, ФИО, группа, программа, год.
2. **Проблема и цель** — цены в маркетплейсах часто меняются; вручную следить
   неудобно. Цель: сервис, который сам отслеживает цену и уведомляет.
3. **Постановка задачи** — список задач (API, БД, парсинг, планировщик,
   уведомления, авторизация, SPA, Docker) и требования.
4. **Выбор инструментов** — схема стека: Go + chi + PostgreSQL (бэкенд),
   Vue 3 + Vite + Chart.js (фронтенд), Telegram Bot API, Docker. Кратко — почему.
5. **Архитектура** — диаграмма из трёх контейнеров (nginx → Go → PostgreSQL),
   горутины планировщика и Telegram-бота.
6. **Модель данных** — ER-схема: users — subscriptions — products —
   price_history — notifications.
7. **Ход решения: парсинг** — интерфейс Parser, разбор JSON Wildberries,
   устойчивость к смене формата.
8. **Ход решения: уведомления** — детект пересечения целевой цены, привязка
   Telegram (/start <код>), отправка сообщения.
9. **Демонстрация (скриншоты)** — дашборд, страница товара с графиком,
   уведомление в Telegram.
10. **Тестирование** — юнит-тест парсера + таблица функциональных проверок.
11. **Результаты** — что сделано, всё работает, запуск одной командой.
12. **Выводы** — закреплённые навыки, направления развития.

**Совет:** на каждый слайд — минимум текста, больше схем/скриншотов; говорить
по 30–60 секунд на слайд.
